Java Environment Setup

Java programming စလုပ်ဖို့ လိုအပ်တဲ့ tools တွေ install လုပ်နည်း၊ environment configure လုပ်နည်း၊ IDE setup လုပ်နည်းကို step-by-step လေ့လာကြမယ်။

1. ဘာတွေ လိုအပ်လဲ? (What You'll Need)

Before you can start writing Java programs, you need to set up your development environment. This involves installing and configuring several essential tools.

☕

JDK

Java Development Kit - Compile & Run Java programs

🔧

Environment Variables

Configure system PATH for command-line access

💻

IDE / Text Editor

IntelliJ IDEA, Eclipse, or VS Code for coding

စတင်မယ့်သူတွေ အတွက်

Java programming ရေးဖို့ အခြေခံ ၃ ခု လိုပါတယ်:

  • JDK (Java Development Kit): Java compiler (javac) နဲ့ JVM ပါဝင်တဲ့ complete package ပါ။ ဒါမှ Java code တွေ compile လုပ်ပြီး run လို့ ရမှာပါ။
  • Environment Variables: Computer ကို JDK ဘယ်မှာ install လုပ်ထားလဲ ပြောပေးရပါတယ်။ အဲ့ဒါမှ command line (Terminal/CMD) ကနေ javac, java commands တွေ သုံးလို့ ရပါမယ်။
  • IDE (Integrated Development Environment): Code editor, debugger, compiler စတာတွေ စုစည်းထားတဲ့ software ပါ။ Notepad လို simple editor မဟုတ်ဘဲ၊ professional development အတွက် ဖြစ်ပါတယ်။

2. JDK, JRE, JVM ဆိုတာ ဘာတွေလဲ?

Before installing, let's understand the difference between JDK, JRE, and JVM:

Java Platform Components 
┌─────────────────────────────────────────┐
│            JDK (Development Kit)         │
│  ┌───────────────────────────────────┐  │
│  │         JRE (Runtime)              │  │
│  │  ┌─────────────────────────────┐  │  │
│  │  │    JVM (Virtual Machine)    │  │  │
│  │  │   - Executes bytecode       │  │  │
│  │  │   - Memory management       │  │  │
│  │  └─────────────────────────────┘  │  │
│  │  + Core Libraries (java.lang,    │  │
│  │    java.util, java.io, etc.)     │  │
│  └───────────────────────────────────┘  │
│  + Development Tools:                   │
│    - javac (Compiler)                   │
│    - javadoc (Documentation)            │
│    - jar (Archive tool)                 │
│    - debugger                           │
└─────────────────────────────────────────┘
အသေးစိတ် ရှင်းပြချက်

1. JVM (Java Virtual Machine)

JVM က အခြေခံဆုံး component ပါ။ Java bytecode (.class files) ကို execute လုပ်တဲ့ virtual machine ပါ။

  • Bytecode ကို Machine Code အဖြစ် ပြောင်းပေးတယ်: Platform-specific machine code အဖြစ် ပြောင်းလို့ program က အလုပ်လုပ်ပါတယ်
  • Memory Management: Garbage Collection နဲ့ memory ကို အလိုအလျောက် စီမံပေးတယ်
  • Security: Bytecode verification လုပ်ပြီး malicious code တွေကို ကာကွယ်ပေးတယ်

📌 JVM က platform-specific ဖြစ်ပါတယ်။ Windows JVM, Mac JVM, Linux JVM တစ်ခုချင်းစီ ရှိပါတယ်။

2. JRE (Java Runtime Environment)

JRE = JVM + Core Libraries (java.lang, java.util, etc.)

  • Java programs run လုပ်ဖို့ လိုတယ်: Already compiled ဖြစ်နေတဲ့ .class files တွေကို execute လုပ်လို့ ရပါတယ်
  • Development မလုပ်ဘူးဆို JRE လုံလောက်တယ်: End users တွေက JRE တင် install လုပ်ရုံနဲ့ Java apps run လို့ ရပါပြီ
  • Core Libraries: String, ArrayList, File I/O စတဲ့ built-in classes တွေ ပါဝင်ပါတယ်

3. JDK (Java Development Kit)

JDK = JRE + Development Tools (javac, javadoc, jar, debugger)

  • Java programs develop လုပ်ဖို့ လိုတယ်: Source code ရေးပြီး compile လုပ်ချင်ရင် JDK လိုပါတယ်
  • javac compiler: .java files တွေကို .class bytecode အဖြစ် compile လုပ်ပေးတယ်
  • Development tools: javadoc (documentation generator), jar (package tool), jdb (debugger) စတာတွေ ပါပါတယ်
  • JRE ပါပါတယ်: JDK install လုပ်ရင် JRE နဲ့ JVM အလိုအလျောက် ပါပါတယ်

📊 ဘယ်အချိန်မှာ ဘာသုံးရမလဲ?

  • Developer (Program ရေးမယ်): JDK install လုပ်ပါ
  • End User (Program run ဖြစ်မယ်): JRE install လုပ်ပါ
  • Server (Java apps host လုပ်မယ်): JRE လုံလောက်ပါတယ် (production မှာ)

3. JDK Installation (JDK Install လုပ်နည်း)

Step 1: Download JDK

There are several JDK distributions available. We recommend Oracle JDK or OpenJDK:

🔗 Download Links

  • Oracle JDK: https://www.oracle.com/java/technologies/downloads/
  • OpenJDK: https://jdk.java.net/ (Open source, free)
  • Adoptium (Eclipse Temurin): https://adoptium.net/ (Recommended for beginners)

Recommended Version: JDK 17 LTS or JDK 21 LTS (Long Term Support)

JDK Versions အကြောင်း

Oracle JDK vs OpenJDK:

  • Oracle JDK: Oracle ရဲ့ official version ဖြစ်ပြီး commercial license ပါပါတယ်။ Production use မှာ license fee ပေးရနိုင်ပါတယ်။ Performance နဲ့ support ကောင်းပါတယ်။
  • OpenJDK: Open source, အခမဲ့ ဖြစ်ပြီး community-driven ပါ။ Oracle JDK နဲ့ functionality အတူတူပါ။ Personal projects, learning အတွက် သင့်တော်ပါတယ်။
  • Adoptium (Eclipse Temurin): OpenJDK ရဲ့ prebuilt binaries တွေ ဖြစ်ပြီး long-term support ရပါတယ်။ Beginners အတွက် အကောင်းဆုံး ရွေးချယ်မှု ဖြစ်ပါတယ်။

LTS (Long Term Support) Versions:

  • Java 8: အရင်က အများဆုံး သုံးတယ် (2014-2030 support)
  • Java 11: Modern features များပါပြီး stable (2018-2027 support)
  • Java 17: Latest LTS, recommended for new projects (2021-2029 support)
  • Java 21: Newest LTS with cutting-edge features (2023-2031 support)

📌 Beginners တွေ အတွက် Java 17 LTS သို့မဟုတ် Java 21 LTS ကို recommend လုပ်ပါတယ်။

Step 2: Installation Process

🪟 Windows Installation

Windows Installation Steps 
1. Download JDK installer (.exe file)
2. Double-click the installer
3. Click "Next" through installation wizard
4. Note the installation path (e.g., C:\Program Files\Java\jdk-17)
5. Click "Install"
6. Wait for installation to complete
7. Click "Close"

Default Path: C:\Program Files\Java\jdk-17

🍎 macOS Installation

macOS Installation Steps 
1. Download JDK installer (.dmg file)
2. Double-click the .dmg file
3. Double-click the package icon
4. Follow the installation wizard
5. Enter admin password if prompted
6. Click "Install"

Default Path: /Library/Java/JavaVirtualMachines/jdk-17.jdk

🐧 Linux Installation (Ubuntu/Debian)

Terminal Commands 
# Update package list
sudo apt update

# Install OpenJDK 17
sudo apt install openjdk-17-jdk

# Verify installation
java -version
javac -version
Installation Tips
  • Installation path ကို မှတ်ထားပါ: Environment variables set လုပ်တဲ့ အခါ သုံးရမှာ ဖြစ်ပါတယ်
  • Admin rights လိုပါတယ်: Windows/Mac မှာ administrator privileges ရှိရပါမယ်
  • Previous versions: Old Java versions ရှိရင် uninstall မလုပ်လည်း ရပါတယ်၊ multiple versions သုံးလို့ ရပါတယ်
  • Disk space: JDK က approximately 300-500 MB ခန့် လိုပါတယ်

4. Setting Environment Variables (Environment Variables Configure လုပ်နည်း)

After installing JDK, you need to set up environment variables so your system knows where to find Java tools.

Why Environment Variables?

ဘာကြောင့် လိုအပ်လဲ?

Environment variables ကို set မလုပ်ရင် command line (CMD, Terminal) ကနေ javac နဲ့ java commands တွေ သုံးလို့ မရပါ။

Example: javac HelloWorld.java လို့ ရိုက်လိုက်ရင် 'javac' is not recognized as an internal or external command ဆိုတဲ့ error ပေါ်ပါတယ်။

Environment variables set လုပ်ပြီးမှ system က JDK ရဲ့ bin folder ထဲက javac.exe/java.exe တွေကို ရှာတွေ့ပြီး command တွေ အလုပ်လုပ်ပါတယ်။

JAVA_HOME: JDK installation directory ကို ညွှန်ပြပေးတာ ဖြစ်ပါတယ်

PATH: Executable files (javac.exe, java.exe) တွေ ရှိတဲ့ folder ကို ညွှန်ပြပေးတာ ဖြစ်ပါတယ်

🪟 Windows - Setting Environment Variables

Windows Steps 
Step 1: Open System Properties
  - Right-click "This PC" or "My Computer"
  - Click "Properties"
  - Click "Advanced system settings"
  - Click "Environment Variables" button

Step 2: Set JAVA_HOME
  - Under "System variables", click "New"
  - Variable name: JAVA_HOME
  - Variable value: C:\Program Files\Java\jdk-17
  - Click "OK"

Step 3: Update PATH
  - Under "System variables", find "Path"
  - Click "Edit"
  - Click "New"
  - Add: %JAVA_HOME%\bin
  - Click "OK" on all windows

Step 4: Verify
  - Open NEW Command Prompt (CMD)
  - Type: java -version
  - Type: javac -version

🍎 macOS - Setting Environment Variables

Terminal Commands 
# Open terminal and edit bash profile
nano ~/.bash_profile

# Add these lines (press 'i' to insert):
export JAVA_HOME=/Library/Java/JavaVirtualMachines/jdk-17.jdk/Contents/Home
export PATH=$JAVA_HOME/bin:$PATH

# Save and exit (press Esc, then type :wq)

# Reload profile
source ~/.bash_profile

# Verify
java -version
javac -version

🐧 Linux - Setting Environment Variables

Terminal Commands 
# Edit bashrc file
nano ~/.bashrc

# Add these lines at the end:
export JAVA_HOME=/usr/lib/jvm/java-17-openjdk-amd64
export PATH=$JAVA_HOME/bin:$PATH

# Save and exit (Ctrl+X, then Y, then Enter)

# Reload
source ~/.bashrc

# Verify
java -version
javac -version

Verification (အတည်ပြုခြင်း)

Command Line 
java -version
Expected Output 
java version "17.0.9" 2023-10-17 LTS
Java(TM) SE Runtime Environment (build 17.0.9+11-LTS-201)
Java HotSpot(TM) 64-Bit Server VM (build 17.0.9+11-LTS-201, mixed mode)
Command Line 
javac -version
Expected Output 
javac 17.0.9
Troubleshooting

Error: 'java' is not recognized:

  • Environment variables မှန်ကန်စွာ set လုပ်ထားခြင်း ရှိ/မရှိ စစ်ပါ
  • Command Prompt/Terminal ကို close ပြီး ပြန်ဖွင့်ပါ (changes ကို reload လုပ်ဖို့)
  • Computer restart လုပ်ပါ (Windows မှာ တစ်ခါတစ်ရံ လိုပါတယ်)
  • PATH variable မှာ %JAVA_HOME%\bin ပါဝင်ခြင်း စစ်ပါ

Multiple Java versions installed:

  • JAVA_HOME က အသစ်ဆုံး version ကို point လုပ်ထားပါ
  • PATH မှာ JAVA_HOME\bin က အရှေ့ဆုံးမှာ ရှိအောင် ထားပါ

5. IDE Setup (IDE ရွေးချယ်ပြီး Install လုပ်နည်း)

An Integrated Development Environment (IDE) provides tools for writing, compiling, running, and debugging Java code all in one place.

💡

IntelliJ IDEA

Recommended for beginners

Intelligent code completion, refactoring, debugging. Free Community Edition available.

Download: https://www.jetbrains.com/idea/

🌙

Eclipse IDE

Popular & Free

Mature, stable, extensive plugin ecosystem. Completely free and open source.

Download: https://www.eclipse.org/

📝

VS Code

Lightweight

Fast, modern editor with Java extensions. Good for small projects.

Download: https://code.visualstudio.com/

ဘယ် IDE ကို ရွေးမလဲ?

IntelliJ IDEA (Highly Recommended):

  • Pros: Smart code completion, excellent refactoring tools, built-in Maven/Gradle support, great debugger, modern UI
  • Cons: Resource-intensive (needs good RAM), Community Edition မှာ enterprise features တချို့ ပိတ်ထားတယ်
  • Best for: Beginners to professionals, large projects, Spring/Java EE development

Eclipse IDE:

  • Pros: Completely free, large plugin ecosystem, good for enterprise Java, lighter than IntelliJ
  • Cons: UI နည်းနည်း outdated, setup လုပ်ရတာ ရှုပ်နိုင်တယ်
  • Best for: Students, budget-conscious developers, enterprise projects

VS Code:

  • Pros: Very fast, modern, lightweight, good for multiple languages, free extensions
  • Cons: Java support က extension dependent, IDE features နည်းတယ်
  • Best for: Simple projects, quick edits, developers who prefer lightweight editors

📌 Recommendation: Beginners တွေ အတွက် IntelliJ IDEA Community Edition ကို အကြံပြုပါတယ်။ Setup လွယ်ပြီး beginner-friendly ဖြစ်ပါတယ်။

Installing IntelliJ IDEA

Installation Steps 
1. Go to https://www.jetbrains.com/idea/download/
2. Download "Community Edition" (Free)
3. Run the installer
4. Choose installation directory
5. Create desktop shortcut (recommended)
6. Select "Create associations" for .java files
7. Click "Install"
8. Launch IntelliJ IDEA
9. Choose UI theme (Light/Dark)
10. Skip or install featured plugins
11. Click "Start using IntelliJ IDEA"

Creating Your First Project in IntelliJ

Project Setup Steps 
1. Click "New Project"
2. Select "Java" from left sidebar
3. Choose Project SDK (select installed JDK)
   - If not listed, click "Add SDK" → "JDK"
   - Browse to JDK installation folder
4. Name your project (e.g., "JavaLearning")
5. Choose project location
6. Click "Create"
7. IDE opens with project structure
8. Right-click "src" folder → New → Java Class
9. Name the class (e.g., "HelloWorld")
10. Write your code
11. Click green "Run" button
IntelliJ Tips
  • Auto-import: Alt+Enter (Windows/Linux) or Option+Enter (Mac) က missing imports တွေ automatically add လုပ်ပေးတယ်
  • Code completion: Ctrl+Space က suggestions ပေးတယ်
  • Run code: Shift+F10 (Windows/Linux) or Ctrl+R (Mac)
  • Format code: Ctrl+Alt+L (Windows/Linux) or Cmd+Option+L (Mac)
  • Refactor/Rename: Shift+F6 က variable/class names rename လုပ်လို့ ရတယ်

6. Testing Your Setup (Setup မှန်/မမှန် စမ်းသပ်ခြင်း)

Let's verify everything is working by creating and running a simple Java program from command line:

Step 1: Create a Java File

Open any text editor (Notepad, TextEdit, nano) and create a file named Test.java:

Java - Test.java 
public class Test {
    public static void main(String[] args) {
        System.out.println("Java setup successful!");
        System.out.println("JDK version: " + System.getProperty("java.version"));
        System.out.println("OS: " + System.getProperty("os.name"));
    }
}

Step 2: Compile the Program

Open Command Prompt/Terminal, navigate to the folder containing Test.java, and compile:

Command Line 
javac Test.java

If successful, this creates Test.class file (no output = success). If there are errors, check for typos.

Step 3: Run the Program

Command Line 
java Test
Expected Output 
Java setup successful!
JDK version: 17.0.9
OS: Windows 10
အရေးကြီးတဲ့ မှတ်ချက်များ
  • javac Test.java - Compile လုပ်တဲ့ command, .java file name နဲ့ သုံးပါ
  • java Test - Run လုပ်တဲ့ command, .class မပါဘဲ class name တင် သုံးပါ
  • File name နဲ့ class name က exactly တူရပါမယ် (case-sensitive)
  • Command prompt/terminal ကို Test.java file ရှိတဲ့ folder မှာ ဖွင့်ရပါမယ်
  • Compile အောင်မြင်ရင် output မရှိပါ၊ error ရှိမှ message ပေါ်ပါတယ်

7. Key Takeaways (မှတ်သားရမည့် အချက်များ)

  • JDK = JRE + Development Tools: Developers need JDK, end users need JRE
  • JVM: Executes bytecode, platform-specific, enables WORA
  • Java 17 LTS or Java 21 LTS: Recommended versions for new learners
  • Environment Variables: JAVA_HOME points to JDK, PATH includes bin folder
  • Verify Installation: java -version and javac -version
  • IntelliJ IDEA Community: Best IDE for beginners (free)
  • Compile: javac FileName.java creates .class file
  • Run: java ClassName executes the program
  • Command line vs IDE: Both work, IDE is more convenient
  • File name = Class name: Must match exactly (case-sensitive)
